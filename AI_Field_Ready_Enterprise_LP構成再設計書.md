# AI Field Ready Enterprise LP構成再設計書

## 1. LP再設計の前提

本LPは、従来のSES営業支援・人材提案支援ではなく、**事業会社が社内でAI人材を育成し、自社業務課題からAIプロダクト/AI業務改善施策を生み出せる状態を作る教育・伴走サービス**として再設計する。

旧コンセプトで重視していた以下の要素はLP上では扱わない。

- SES営業支援
- スキルシート補強
- 待機人材の案件化
- 外部案件への提案サマリー
- 求人/案件マッチング
- 個人向け学習SNS

新しいLPで訴求すべき中心価値は以下である。

> AIを学ぶ研修ではなく、  
> 自社の業務課題をAIテーマに変換し、PoC設計・プロトタイプ・評価・実装計画まで進められる社内AI人材を育てる。

---

## 2. LPの目的

### 2.1 主目的

事業会社の経営層、DX推進部門、人材開発部門、事業部門、情報システム部門から、以下の問い合わせを獲得する。

- 社内AI人材育成について相談したい
- 自社業務課題をAIテーマ化できるか相談したい
- 12週間カリキュラムの詳細を確認したい
- PoCにつながる実践型研修を検討したい
- 部門横断のAI人材育成プログラムを導入したい

### 2.2 CVポイント

| CV | 目的 | 優先度 |
|---|---|---:|
| 無料相談を申し込む | 商談化 | 高 |
| AI人材育成診断を受ける | リード獲得・課題把握 | 高 |
| カリキュラム資料をダウンロード | リード獲得 | 高 |
| サンプル成果物を見る | 価値理解 | 中 |
| 12週間プログラムの説明を受ける | 具体検討 | 中 |

### 2.3 メインCTA

LP全体でCTAは以下の3つに整理する。

| CTA | 用途 | 表示場所 |
|---|---|---|
| AI人材育成診断を相談する | Primary CTA | Hero/Header/Sticky/Final CTA |
| カリキュラム資料を見る | Secondary CTA | Hero/Header/中盤/Final CTA |
| サンプル成果物を見る | Tertiary CTA | 成果物セクション/Product Demo |

「ログイン」はLPの右上に小さく配置し、主導線から外す。

## 3. ターゲット別訴求整理

### 3.1 主要ターゲット

| ターゲット | 課題 | 刺さる訴求 |
|---|---|---|
| 経営層 | AI活用方針はあるが、社内から実行テーマが出てこない | AI活用を研修で終わらせず、PoC候補と実装判断材料まで作る |
| DX推進部門 | 各部門のAI活用テーマを整理・推進できない | 部門課題をAIテーマ化し、PoC計画まで伴走する |
| 人材開発部門 | AI研修の成果が業務成果につながらない | 受講完了ではなく、業務課題定義書・PoC計画書を成果物にする |
| 事業部門 | 生成AIを使いたいが、自部門業務への落とし込みが難しい | 自部門の課題を題材に、実際に使えるAI施策を設計する |
| 情報システム部門 | セキュリティや運用を考慮しないAI利用が広がる | ガバナンス、データ制約、運用設計まで含めて育成する |

### 3.2 除外・弱めるターゲット

| ターゲット | 理由 |
|---|---|
| 個人学習者 | B2C化し、競争環境が悪化するため |
| SES企業 | 今回の方向転換により主対象から外すため |
| 転職希望者 | マッチング要素を廃止するため |
| 汎用AIリテラシー研修だけを求める企業 | 差別化が弱く、価格競争になりやすいため |

## Section 03. Problem：事業会社がAI人材育成で直面する課題

### 目的
「AI研修を導入したが成果につながらない」という顧客課題を明確化する。

### 見出し

```text
AI研修を実施しても、現場のAIプロジェクトが生まれない理由
```

### 課題カード

| 課題 | 説明 |
|---|---|
| 研修がAIリテラシーで止まる | 生成AIの使い方は学んでも、自社業務のどこに適用すべきか設計できない |
| 各部門からAIテーマが出てこない | 現場課題、KPI、データ状況、制約を整理する型がない |
| PoC計画に落とし込めない | アイデアは出るが、検証範囲、成功条件、評価指標、体制が曖昧なまま止まる |
| 情シス/法務/現場の共通言語がない | セキュリティ、データ利用、運用、リスクの観点が後回しになる |
| 研修成果が経営判断に使えない | 受講完了率やテストスコアだけでは、次の投資判断につながらない |

### Before/After表現

```text
Before：生成AI研修を受けたが、現場業務への適用テーマが出てこない
After：各部門から、PoC計画書・評価指標・実装ロードマップつきのAIテーマが出てくる
```

## Section 04. Solution：AI Field Ready Enterpriseとは

### 目的
サービスの全体像を短く説明する。

### 見出し

```text
業務課題を、AIプロジェクト候補へ変える実践型カリキュラム
```

### 3つの提供価値

| 提供価値 | 内容 |
|---|---|
| 業務課題起点 | 自社の業務フロー、KPI、データ、制約を題材にするため、研修が現場課題に直結する |
| 成果物駆動 | 受講完了ではなく、業務課題定義書、AIテーマ企画書、PoC計画書、評価レポートを作成する |
| プロジェクト化支援 | 研修後に、PoC候補、実装ロードマップ、運用設計、経営提案へ接続する |

### 補足コピー

```text
AIを「使える人」を増やすだけでは不十分です。
必要なのは、自社の業務課題を見極め、AIで解くべきテーマを選び、検証計画まで作れる人材です。
```

## Section 06. Product / Program Demo：カリキュラム体験デモ

### 目的
実際にどのような流れで成果物が作られるかを見せる。

### 見出し

```text
業務課題からPoC計画まで、カリキュラム内で作り切る
```

### タブ構成

| タブ | 見せる内容 |
|---|---|
| 業務課題登録 | As-Is業務、課題、KPI、関係者、制約を入力する画面 |
| AIテーマ化 | RAG/AI-OCR/データ分析/エージェント/自動化に分類する画面 |
| データ棚卸し | 利用可能データ、文書、アクセス権、品質、更新頻度を整理する画面 |
| PoC計画 | 仮説、検証範囲、成功条件、評価指標、体制を設計する画面 |
| 成果物レビュー | AIレビュー、メンターコメント、改善履歴、承認状態を表示する画面 |

### 表示例：業務課題登録

```text
対象部門：カスタマーサポート
課題：問い合わせ一次回答に時間がかかる
現行KPI：平均初回回答時間 8時間
改善目標：初回回答時間を30%削減
利用可能データ：FAQ、製品マニュアル、過去問い合わせ履歴
制約：個人情報を含む問い合わせ文の取り扱い
```

### 表示例：AIテーマ化

```text
推奨AI活用テーマ：RAG/社内ナレッジ検索
理由：FAQ・マニュアル・問い合わせ履歴が存在し、回答根拠の提示が重要なため
PoC候補：問い合わせ回答支援AI
想定成果：回答作成時間削減、回答品質の平準化、教育工数削減
```

## Section 10. Review System：AIレビュー＋メンターレビュー

### 目的
成果物の品質が担保される仕組みを説明する。

### 見出し

```text
提出して終わりではなく、実務品質までレビューする
```

### レビュー構造

| レビュー | 内容 |
|---|---|
| AIレビュー | テンプレートの抜け漏れ、論理構成、評価指標、リスク記載を一次確認 |
| メンターレビュー | 実務観点で、業務適合性、PoC実現性、評価設計、運用性を確認 |
| 改善履歴 | 再提出、改善コメント、承認状態を記録 |
| 成果物承認 | PoC候補や経営提案に使える水準かを判定 |

### ルーブリック例

| 評価軸 | 評価内容 |
|---|---|
| 業務課題の具体性 | 課題、対象業務、KPI、関係者が具体化されているか |
| AI適用の妥当性 | AIで解くべき理由と、人が担うべき部分が整理されているか |
| データ準備度 | 利用データ、品質、権限、更新頻度が把握されているか |
| PoC設計力 | 検証範囲、成功条件、評価指標が明確か |
| ガバナンス | セキュリティ、個人情報、運用リスクが考慮されているか |
| 提案力 | 経営/部門が判断できる資料になっているか |

## 13. LPで最も強く伝えるべきこと

このLPで一番伝えるべきことは、以下である。

> AI Field Ready Enterpriseは、AIを学ぶだけの研修ではない。  
> 自社の業務課題をAIテーマに変換し、PoC計画・評価レポート・実装ロードマップまで作成できる社内AI人材を育てるプログラムである。

そのため、LPの主役は「教材一覧」ではなく、以下の成果物にする。

- 業務課題定義書
- AIテーマ候補リスト
- データ/文書棚卸し表
- PoC計画書
- 評価レポート
- AIプロジェクト提案書
- 実装ロードマップ

これらを大きく見せることで、一般的なAI研修やeラーニングとの差別化が明確になる。
