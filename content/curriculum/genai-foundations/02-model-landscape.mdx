# 主要サービスと使い分け

生成AIサービスは多数ありますが、業務導入では「UI製品」と「API」を分けて考えると整理できます。

## 代表的なLLMと特徴

- ChatGPT（OpenAI / GPT系）: 汎用性が高く、エコシステムが広い
- Claude（Anthropic）: 長い文脈の取り扱いと、丁寧な指示追従に強み
- Gemini（Google）: Googleサービスとの連携、マルチモーダル対応

どれも「次トークン予測」という基本は同じです。違いは、文脈長・コスト・速度・得意なタスク・周辺機能です。1つに固定せず、タスクで使い分ける前提を持つと安全です。

## UI製品 と API の違い

- UI製品（チャット画面）: 個人の試行錯誤・下書きに最適。手早い。
- API: アプリやワークフローに組み込み、再現性のある自動化を作れる。

業務を「仕組み化」するなら、最終的にはAPIで組み込むことになります。本コースの後半はAPIを意識したコードを書きます。

## トークンとコストの感覚

LLMの入出力は「トークン」で課金されます（おおよそ日本語1文字が1〜2トークン）。

- 入力（プロンプトと文脈）と出力の両方がコストになる
- 長い文書を毎回丸ごと渡すと高コスト・低速になる
- だから「必要な部分だけ渡す」設計（RAG・要約）が重要になる

## モデル選定の実務チェック

- タスクの難易度に対してモデルが過剰または不足でないか
- 機密データを外部に送ってよいか（ガバナンス・契約条件）
- 必要な文脈長・応答速度・コストのバランス
- 失敗時の影響度（人手レビューを挟むかどうか）
