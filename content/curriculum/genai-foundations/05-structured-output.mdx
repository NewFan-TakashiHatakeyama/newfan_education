# 生成AIの出力を構造化データにする

生成AIをアプリに組み込むには、自然文の出力を「プログラムが扱えるデータ」に変換する必要があります。ここでは「キー: 値」形式の出力を辞書に変換します。

## なぜ構造化が必要か

- 後続処理（DB保存・集計・分岐）には構造化データが必要
- だからプロンプトで出力形式を固定し、コードでパースする
- 形式が崩れたときの検証（必須キーの有無）も実務では重要

## このレッスンの演習

`parse_fields(text)` を実装し、複数行の「キー: 値」テキストを辞書に変換します。さらに必須キー（例: amount）が含まれるかを確認します。

## 実装のヒント

- 行ごとに分割し、最初の「:」で key と value に分ける
- 前後の空白を取り除く
- 必須キーの存在チェックを入れて、欠損に気づけるようにする

## 現場での使いどころ

- 請求書・申込書などの抽出結果を業務システムに連携する
- 出力形式を固定し、崩れたら検知して人手レビューに回す
