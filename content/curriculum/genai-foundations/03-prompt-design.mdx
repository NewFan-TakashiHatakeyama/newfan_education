# プロンプト設計の基本

プロンプトは「AIへの仕様書」です。良いプロンプトは、役割・文脈・指示・制約・出力形式を明確にします。

## 良いプロンプトの5要素

- 役割（Role）: 「あなたは〇〇の専門家です」
- 文脈（Context）: 前提・背景・対象データ
- 指示（Instruction）: してほしいことを具体的に
- 制約（Constraints）: 文字数・観点・禁止事項
- 出力形式（Format）: 箇条書き / JSON / 表 など

## Few-shot（例示）で精度を上げる

期待する入出力の例を1〜3個見せると、形式とトーンが安定します。分類や抽出のタスクで特に効果的です。

- 例を「入力 → 期待出力」のペアで示す
- 例は実際のデータに近いものを選ぶ
- 出力形式を例で固定する（後段でパースしやすくなる）

## よくある失敗と対策

- あいまいな指示 → 具体的な動詞・観点・件数を指定する
- 長すぎる文脈 → 必要部分に要約・抽出してから渡す
- 出力が崩れる → 「JSONのみを出力」など形式を強制し、例を添える
- ハルシネーション → 「文脈に無いことは『不明』と答える」と明示する

## 業務プロンプトの型（テンプレート）

実務では、変数だけ差し替えられるテンプレート化が有効です。次のレッスンでは、この型をPythonの関数として実装します。

- 役割・文脈・質問・出力形式を引数で受け取る
- 同じ品質のプロンプトを誰でも再現できるようにする
